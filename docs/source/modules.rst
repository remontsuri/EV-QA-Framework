Module Overview
===============

The ``ev_qa_framework`` package is organized into the following modules:

Core Modules
------------

.. list-table::
   :header-rows: 1

   * - Module
     - Description
   * - ``soh_predictor``
     - State-of-Health prediction using ML models
   * - ``analysis``
     - Telemetry analysis and anomaly detection
   * - ``battery_scoring``
     - Battery health scoring algorithms
   * - ``physics_features``
     - Physics-based feature engineering
   * - ``digital_twin``
     - Digital twin simulation for battery packs
   * - ``fleet_analytics``
     - Fleet-level analytics and aggregation

Advanced Modules
----------------

.. list-table::
   :header-rows: 1

   * - Module
     - Description
   * - ``automl``
     - AutoML pipeline for model selection
   * - ``soh_transformer``
     - Transformer-based SOH prediction
   * - ``v2g_scenarios``
     - Vehicle-to-Grid scenario simulation
   * - ``hil``
     - Hardware-in-the-Loop testing support

Infrastructure Modules
----------------------

.. list-table::
   :header-rows: 1

   * - Module
     - Description
   * - ``can_bus``
     - CAN bus communication layer
   * - ``dbc_parser``
     - DBC file parser for CAN databases
   * - ``config``
     - Configuration management
   * - ``cli``
     - Command-line interface
   * - ``models``
     - Data models and schemas
   * - ``metrics``
     - Metrics collection and reporting
